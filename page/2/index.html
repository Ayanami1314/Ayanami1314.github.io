git初步
2023/10/16

﻿

  1. git bash

  2. $git clone +url 下载别人的项目

  3. $git init 在当前目录文件夹新建一个git仓库

  4. $git add name （前提有仓库）跟踪某个文件

  5. $git rm name 删除

  6. $git rm –cache name 不被跟踪但不删除

  7. 修改完一个文件 再次$git add file-name 把它设置为缓存状态

  8. $git reset HEAD file-name 取消缓存状态

  9. 通过$git commit 提交此次修改

  10. $ git commit -m “对本次修改的一些注释”

  11. $git status 看文件状态

  12. $git diff 查看细节的修改位置

  13. $git log 查看提交历史

  14. 远程仓库 remote
    $ git remote add name(你起的名字) +url
    $ git remote rename …改名
    $git push 仓库名字 分支， 提交代码（需要github令牌）

另一种：ssh协议，相对复杂

  1. 分支，存放git对象的一个哈希值，类似指向不同版本文件的指针
    初始会自动新建分支master

  2. 查看分支 $git branch –list

pipeline
2023/10/16

﻿## 一、为什么要引入流水线
pipeline: 同时执行多条指令
每一条指令的执行可以分解成多个阶段，在RISC-V之中为IF, ID, EXE, MEM和WB，即

  • instruction fetch
  • instruction decode(regfile op & imm-gen)
  • execution(alu and branch-compare)
  • memory operation
  • write back(to regfile)
    下图是一个单周期的数据路径

有一种提高效率的思想叫流水线pipeline, 在上图之中，一条指令执行的过程中alu只在EXE阶段工作，其他阶段不工作；regfile只在ID阶段工作，其他阶段也不工作。既然要提升效率，自然想到，如何让alu、regfile等所有原件一直工作？类似工厂流水线的模式，alu计算完这一条指令之后立刻计算下一条，而不是等待这一条指令执行完再算。

python并发
2023/10/16

﻿并发：多协程、多线程、多进程、多机器

任务：IO密集和cpu密集
IO密集：数据库、文件读写等
cpu密集： 计算密集

多线程：单cpu，目的是io和cpu分离并行执行，适用于io密集，内存开销比多进程少

多进程：多cpu，目的是多cpu并行计算，适用于cpu密集，内存开销大

线程threading库

t = threading.Thread(target=func, args=(…))
t.start() # 开始线程，执行func
t.join() # 等待线程结束

for _ in range(count):
    thread = threading.Thread(target = f, args=(...))
