    int* pMazeState; // 配合下面的 enum使用一个int记录每个各自的连接和填充状态

        cell_path_EAST=0x01, //0000 0001

python并发
2023/10/16

﻿

并发：多协程、多线程、多进程、多机器

任务：IO密集和cpu密集
IO密集：数据库、文件读写等
cpu密集： 计算密集

多线程：单cpu（前提：python GIL锁），目的是io和cpu分离并行执行，适用于io密集，内存开销比多进程少

多进程：多cpu，目的是多cpu并行计算，适用于cpu密集，内存开销大

线程threading库

t = threading.Thread(target=func, args=(…))
t.start() # 开始线程，执行func
t.join() # 等待线程结束

例如：

threads = []
for _ in range(count):
    thread = threading.Thread(target = f, args=(...))
    threads.append(thread)
for thread in threads:
    thread.start()
for thread in threads:
    thread.join()

如果多个线程之间存在共享资源
需要加锁，
lock = threading.Lock()给需要线程独占的运行函数f套上with lock:

当对一个类的某个方法加锁的时候，需要在__init__上锁self.lock = threading.Lock()
在方法调用之中
with self.lock：
而不是直接在方法调用之中创建lock, 这样无法保证创建lock与线程切换的顺序

生产者-消费者模型：

以爬虫为例：
生产者线程get网络资源，存储到任务序列之中
消费者线程取出任务序列的值，存储到目标文件

任务序列 queue.Queue
例 

buffer = queue.Queue(maxsize = 10)
def producer():
    item = ...;
    buffer.put(item)
def consumer():
    while(True):
        item = buffer.get()
        # do something

线程池：

from concurrent.futures import ThreadPoolExetutor
with concurrent.futures.ThreadPoolExetutor as pool:

    tasks = [...]
    results = pool.map(func, tasks)
    for res in results:
        print(res)

库函数基本和多线程完全一样
把threading 库换成multiprocess就行
还有就是queue.Queue() 换成 multiprocess.Queue()
