2023.11.6-2023.11.13学习周记

part1: jyy lec

调试理论

核心：程序是一个状态机，其状态是内存M，寄存器R的结合{M，R}，通过每一条指令转移状态

一个错误的程序Bug/fault，在经过一定的指令数之后，出现了理论状态与实际状态不符合的情况（Error）, 此时开始，可以被检测到，再经过一定的指令数（可能很长很长甚至没有）后，程序会崩溃形成一个直接可见的failure

程序状态：

​ |fault |Error |failure

right–>right–>…right–>error–>error–>error–>…

只有在right–>error这一步，才能被gdb调试出来，其他的right–>right, error–>error都是无法使用单步调试调出bug的

如何从failure定位fault?

printf: 抽取程序状态主要部分，可视化

二分printf, 寻找right–>error这一步，找到error处，进而回溯fault

更进一步？避免二分的麻烦

写好充足的assert等，让Error尽早变成failure

放到pa之中，即为Assert(), Log(), panic()函数的使用

更进一步：防御性编程

一些技巧：

  1. 使用const，volatile(并行，防止编译优化打乱顺序)，坚持给单模块函数加上static（pa里面体现明显）

  2. 打开所有警告开关 -Wall -Werror

  3. 缓冲区溢出：使用strncpy代替strcpy, -fsantinize=address

  4. 显式初始化所有变量，等等等

IO和显卡

OS眼中的外设：三（或更多）个寄存器+一个程序

三个典型寄存器：外设状态state, 外设交换数据data，外设命令command

外设一般有自己的微型cpu，内存，芯片等，在OS眼中外设只是一些接口（软件上还附带了如何使用这些接口的驱动文件），OS只负责给外设发指令（指令集和发什么指令，什么时候发指令由外设的驱动提供），之后外设自己处理指令，并通过接口告诉OS

两种IO模式：memory-mapped IO & port-mapped IO (MMIO&PMIO)

mmap：在内存中保留一段地址空间用于存放外部设备

外设提供寄存器—>OS将其映射到某个内存地址—>之后如果其它程序要读写外设（当然它们只知道读写那个内存地址）—>OS转发给外设寄存器—>外设执行命令，OS使用polling或者interrupt的方式获取命令结果

OSTEP的伪代码（polling方式）

While (STATUS==BUSY)
    ; // wait until device is not busy 
Write data to DATA REG
Write command to COMMAND REG
    (device do something)
While (STATUS==BUSY)
    ; // wait until device is done

interrupt方式需要在中断表之中添加表项（现代的中断控制器集成在cpu之中，管理中断优先级和屏蔽等）

mmap好处是对硬件要求低，坏处提高了软件代码的编写难度

pmap: 在cpu上开出不同的IO端口

此时，IO的指令不再是和内存相同的write, read，而变成了特殊cpu指令；IO的存储也和内存分开，变成独立的空间

优点：省内存，缺点：耗cpu引脚

典型外设工作分析：

鼠标：（现代光学鼠标）工作原理为，

工作时发光二极管照亮鼠标底部表面，微型摄像头以一定时间间隔不断进行图像拍摄。鼠标在移动过程中产生的不同图像传送给光学引擎进行数字化处理，由定位芯片判断出鼠标的移动方向和距离，分析出来的结果转换成坐标偏移量从而实现光标定位。

省流就是拍照–>算出移动—>发给电脑，乘以鼠标灵敏度—>屏幕光标移动

打印机：

简单打印：ASCII，复杂打印：图片等

一种”编程语言“，描述各个字符、图片的位置、字体、旋转等等等

postscript —-> pdf

打印：cpu发送脚本给打印机，打印机执行

总线：管理IO的IO

”即插即用“是怎么实现的？电脑接口–>电信号—>IO总线，通过某种协议处理—->通知cpu，分配、管理硬件地址、发现设备……

lspci -t lsusb -t可以列出电脑上的总线

显卡：从打印机的视角理解

显卡就是另一种”打印机“

cpu—>发送”脚本“—>显卡执行

（例：cpu—>贴图位置x,y, 方向，前景后景？，使用哪个调色板？—>gpu接受，绘图）

如果让cpu画图：一条指令，一个像素，一次IO（典型情况）

cpu发脚本和素材给gpu, gpu并行计算脚本、画图

素材存在gpu的显存之中 以纹理，如8*8的小贴块的形式

马里奥是如何在几十kb的空间之中被编出来的？

蘑菇：左右移动？水平翻转！

大圆球：实际纹理只有1/4圆，翻转四次

1，2P：同一个贴图，不同调色板

cpu和显卡的本质区别：

cpu是冯诺依曼机，执行任意程序；显卡只做计算-画图这一件事

cpu需要处理强时序问题，并行性有限；显卡各像素独立，可以在电路规模上大幅度并行

2D->3D 多边形三角剖分，三角管线、运算、渲染

lexer: 可以用正则表达式

parser: BST递归下降

表达式求值

什么是表达式

<expr> ::= <operand> // number, register, etc

​ | “(“ <expr> “)”

​ | <expr> <binary-op> <expr> // binary-op: + - * / etc

​ |<prefix-single-op><expr>

​ |<expr><suffix-single-op>

那么对应的求值器就可以这么写

// pseudo code
word_t eval_expr(Token *e, int start, int end){
    if(start==end){
        return token[start];
    }
    if(check_bracket(start, end)){
        return eval_expr(start+1, end-1);
    }
    else{
        int main_op_idx=find_main_op(e, start, end);
        Token* main_op_tk = e + main_op_idx;
        if(main_op_tk.type == BINARY_OP){
            switch(main_op_tk.type){
                case '+': return eval_expr(start, main_op_idx-1) + \
                    eval_expr(main_op_idx+1, end);
                // ... etc      
            }
        }
        else if(main_op_tk.type == SINGLE_PREFIX_OP){
            switch(main_op_tk.type){
                assert(main_op_idx == start + 1);
                case DEREF: return mem[eval_expr(start, main_op_idx-1)];
                // ... etc      
            }
        }
        else if(main_op_tk.type == SINGLE_SUBFIX_OP){
            // ...etc
        }
        //...
    }
}

int find_main_op(Token *e, int start, int end){
    // 括号外，且优先级最高的op是主op
    Token ops[32] = {};
    int main_op_idx = start;
    int max_priority = something;
    Token* cur = e;
    for(cur = e+start;cur!=end;++cur){
        if(cur->type == '('){
            while(cur->type!=')'){
                cur++;
            }
            // 错误处理
        }
        else if(is_op(cur->type) && priority(cur->type)>max_priority){
            main_op_idx = cur - (e+start);
            max_priority = priority(cur->type);
        }
    }
    return main_op_idx;
}
