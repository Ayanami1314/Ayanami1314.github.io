2023.11.6 正则表达式

正则：描述文本匹配的代码

最简单的正则：hi 匹配 hi

如果精确匹配hi，那么需要在前后加上\b，\b代表单词的开头或结尾, 是一个占位符，不代表空格或者换行（前后有一个空格能识别出这是一个单词即可）

\bhi\b.*\Ayanami\b 表示匹配{hi(若干个字符)Ayanami}

大概这样

\bhi\b.*\bayanami

hi,ddd ayanami

.表示匹配任意字符（除了换行），*表示前一个字符.可以重复任意次

\d ：0-9的一位数字

\d\d\d\d-\d\d\d\d\d\d\d\d 0579-12345678

重复简化语法

\d{4}-\d{8}

其他元字符

\s:匹配任意空白符

\w：数字、字母、下划线、（甚至可能支持汉字）

-D\w*=\w*\s

抓取make之中=的预定义宏-D

需要注意在这个时候，如果使用\s或者\b前后包裹起来，由于文本之中的-D可能是连着的，而两个连着的之间的空格只匹配一次，会漏匹配

抓-I

-I[a-zA-Z0-9\/]*

^匹配字符串开始

$匹配字符串结束

5-12位数字

^\d{5,12}$ (如果不用^$只能保证匹配的是数字，但不能保证数字连续)

对于\d{5,12}而言，使用这样的方法就只能保证字符串里包含5到12连续位数字，而不是整个字符串就是5到12位数字。

字符转义

元字符或者\， \\, \*,\.

重复

代码/语法 说明
* 重复零次或更多次
+ 重复一次或更多次
? 重复零次或一次
{n} 重复n次
{n,} 重复n次或更多次
{n,m} 重复n到m次

字符集

[aeiou]匹配a,e,i,o,u

[0-9a-zA-Z]

分支条件, 多种匹配取其一（有截断，先匹配左边）

|分割

\(\d{4}\)-\d{8}|\d{4}-\d{8}

反义：

  • \W：不是字母数字下划线

  • \S: 不是空白字符

  • \D: 不是数字

  • \B：不是单词的开始或结束

  • [^x]：不是x的任意字符

  • [^aeiou] ： 不是aeiou的任意字符

\S+ 不包含空白符的字符串

分组：多字符重复

多字符重复需要将子表达式套上小括号

(\d{1-3}.){3}\d{1-3} 简易ip地址匹配

但存在问题，ip之中三位是0-255

只能用很呆的方法

(2[0-4]\d|25[0-5]|[01]?\d?\d\.){3}(2[0-4]\d|25[0-5]|[01]?\d?\d)

后向引用：

子表达式会拥有组号，从左到右1，2，… 组号0是整体的表达式

后向引用用于重复某种分组匹配, 可以用\1指定第一组，\2指定第二组，etc

例如

\b(\w+)\b\s\1\b

用于匹配shit shit或者oh oh这样的连续重复单词，这里的\1是前面1组匹配的最终结果而不是字面表达式的复制

指定组名

(?”GroupName”\w+) 指定了这个组名为GroupName

引用时使用\k<GroupName>

零宽断言

断言：当断言为假的时候停止匹配

(?=exp) 断言：匹配的位置后面是exp

如：\b\w+(?=ing\b) 捕获singing, dancing等等，匹配结果为sing, danc

(?<=exp)断言：匹配位置的前面是exp

如：(?<=\bpre)\w+\b 捕获prereading 等等，匹配结果为reading

(?!exp)断言：匹配的位置后面不是exp

用于处理[^u]虽然达成了不是u的效果，但是要求必须存在一个字符的side effect

\b\w+\q(?!u)\w*\b 包含字母q，且后面不是u的单词

(?!<exp)断言：匹配的位置前面不是exp

\b(?!<_)usr\w*\b: 匹配usr而不匹配_user

(?<=<(\w+)>).*(?=</\1>): 匹配<html></html>格式

注释：（?#comment）

可以（?#cm1 #cm2 #cm3）

贪婪匹配和懒惰匹配：

通常情况：多是贪婪匹配，即匹配尽可能长的文本

a.*b 匹配aabab时，会匹配aabab全体

有时候我们需要匹配尽可能短的文本（懒惰匹配）

a.*?b 匹配aabab就会匹配aab和ab

懒惰限定符号：

  • *？
  • +？
  • ?? 重复0或1次，但尽量少重复
  • {n, m}?
  • {n,}?

平衡组和递归匹配：

抓取嵌套结构

(?’group’exp) : 把匹配的exp命名为group，并压入堆栈

(?’-group’exp): 匹配到exp时，堆栈上弹出group捕获内容，若栈空，则匹配失败

(?(group)yes|no):如果栈上有group,那么匹配yes，否则匹配no

(?!): 断言，由于没有后续的匹配部分?=exp，exp为空，必然失败

应用，匹配嵌套<div>

< # 最外层左括号

​ [^<>]* # 不是括号的东西：最外层的内容

​ (

​ (?’bracket’<) # 匹配<, 并且命名为braket组, 入栈

​ [^<>]*

​ )+ # 1-n个左括号

​ (

​ (?’-bracket’>) #匹配>,出栈braket

​ [^<>]*

​ )+

​ (?(bracket)(?!)) # 如果bracket栈不空，即左右括号数量不匹配，则执行失败断言(?!)

在c++之中使用正则表达式

c11

#include<iostream>
#include<regex>
#include<string>
using namespace std;
int main(){
    string handle_string="_ARanddomFileName1";
    regex regex_pattern("[A-Za-z0-9_]+[A-Za-z0-9]+");
    // 判断
    if(regex_match(handle_string, regex_pattern)){
        cout<<"matched\n";
    }
    if(regex_match(handle_string.begin(),handle_string.end(), regex_pattern)){
        printf("string:range(0-%d)=>matched\n", handle_string.end()-handle_string.begin());
    }
    // 搜索
    smatch sm;
    regex_search(handle_string, sm, regex_pattern);
    for(auto x: sm){
        cout<<x<<" ";
    }
    // 替换
    regex_replace(handle_string, regexp, "replace_new_word");
    return 0;
}
