10.26 技术随笔

回调函数：

假设有这样的工作流

​ // some code

​ result = do_something（args)

​ // some code related to “result”

​ // some other code

假如这个do_something是一个网络/磁盘IO等需要等待的操作，那么有几种执行的策略

  • 从上到下，卡在do_something这里

  • 暂时不管do_something, 先做some other code, 并polling do_something这个函数，一旦它return 了，就执行some code related to “result”

  • 尝试把some code related to “result” 丢到 do_something之中， 这样外部工作流就与do_something解耦了

显然最后一种较好，但问题是，do_something这个函数，可能是一个库之类，它在编写的时候不知道自己的result会被哪些函数用，怎么办?

只能用户自定义

而策略是将do_something添加一个函数参数callback, 意为do_something这个函数返回之后执行的函数（使用result的some code related to “result” 部分）

变成

​ // some code

​ result = do_something（args， callback)

​ // some other code

而此时注释部分与非注释部分已经可以分离并行，提高了效率

继续看bash

Section 5：bash 变量：

  • 环境变量

    $ printenv
$ env
# 打印所有环境变量
$ printenv PATH
$ echo $PATH
# 打印单个
# 需要注意bash变量区分大小写
$ set
# 显示所有变量，包括自定义变量和bash函数

  • 关于变量，所有的变量都是字符串， 变量可以是命令执行结果

  • $a_file 与${a}_file

  • 可以嵌套变量，USER=Ayanami myvar=USER, echo ${!myvar} 得到的是Ayanami, !展开变量

  • 删除变量， unset 或者赋值为空字符串

export, set, setx的区别

set（bash） 只是设置当前bash进程(不包括子进程)的变量, eg

Ayanami@LAPTOP-5ASA2K3P MINGW64 /d/Desktop/notes
$ echo $name
haha

Ayanami@LAPTOP-5ASA2K3P MINGW64 /d/Desktop/notes
$ bash

Ayanami@LAPTOP-5ASA2K3P MINGW64 /d/Desktop/notes
$ echo $name

export VAR=… 设置的变量能够被子进程识别，但是bash重启时会删除

要永久更改bash的path, 需要更改配置文件

Windows下提供了setx命令，免去了配置文件的更改，相等于直接修改环境变量

setx VAR VAL

特殊变量

$ echo $?  #$?为上一个命令的退出码
$ echo $$ # $$是shell 进程ID
# $_是上一个命令的最后一个参数
# $# 脚本的参数数量，$@脚本参数值

变量默认值

${varname:-word} # return var exist? value: word
${varname:=word} # return var exist? value: word;if !var, value= word;
${varname:+word} # return var exist? empty: word;Test if the var exist
${varname:?message} # var exist? return value: print "varname:{message}" and interrupt the bash script.

# 例
ayanami@LAPTOP-5ASA2K3P:~$ ayanami=kawai
ayanami@LAPTOP-5ASA2K3P:~$ ${aya:?"aya is not ayanami"}
-bash: aya: aya is not ayanami

# 例二
filename=${1:?"filename missing"} # 1 是脚本的第一个参数

declare -r VAR 设定只读变量， -u自动转换大写， -l自动转换小写

declare -p 输出所有变量信息

let 声明，直接执行算术表达式 let var=1+2
