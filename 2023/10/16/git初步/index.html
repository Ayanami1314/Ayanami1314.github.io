git初步

2023/10/16 入门 命令行

﻿

  1. git bash

  2. $git clone +url 下载别人的项目

  3. $git init 在当前目录文件夹新建一个git仓库

  4. $git add name （前提有仓库）跟踪某个文件

  5. $git rm name 删除

  6. $git rm –cache name 不被跟踪但不删除

  7. 修改完一个文件 再次$git add file-name 把它设置为缓存状态

  8. $git reset HEAD file-name 取消缓存状态

  9. 通过$git commit 提交此次修改

  10. $ git commit -m “对本次修改的一些注释”

  11. $git status 看文件状态

  12. $git diff 查看细节的修改位置

  13. $git log 查看提交历史

  14. 远程仓库 remote
    $ git remote add name(你起的名字) +url
    $ git remote rename …改名
    $git push 仓库名字 分支， 提交代码（需要github令牌）

另一种：ssh协议，相对复杂

  1. 分支，存放git对象的一个哈希值，类似指向不同版本文件的指针
    初始会自动新建分支master

  2. 查看分支 $git branch –list
    ‘* ’代表当前分支

  3. 创建分支 git branch + 新分支name
    切换分支 git checkout name
    (可以合并为 git checkout -b name)

  4. git log –all –graph 看结构
    git merge name 将当前分支（head指向的）和name分支合并
    q退出
    （修改同样的地方会merge conflict
    用git status 看冲突的位置，重新编辑）
    vi file_name 用vim 编辑file
    esc+:wq退出保存

  5. $ git fetch 拉取远程仓库的分支
    之后 git checkout -b new_name 仓库名/分支名

  6. git stash 储存，储存了之后可以不提交切换分支
    之后通过 git stash apply 继续编写
    git stash list看所有储存的版本
    git stash apply stash@{2} 指定应用版本2
