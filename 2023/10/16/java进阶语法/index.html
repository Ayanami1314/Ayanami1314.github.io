java 语法和收获：

  1. 随机测试非常好用，对应java就是StdRandom库
  2. java的接口机制 interface 有点类似c++的抽象类，继承接口的需要Override所有的接口函数声明（接口里面只写声明）（最好显式@Override）

通过接口机制，我们可以统一不同的实现，不管你写的是AList, SLList, 还是LIST61B, 只要他们继承于同一个接口，对于使用者来说就是好的

一个更有意义的地方是，接口保证了java编译时的静态编译（或者简洁性）：
java不需要知道一个类的内部实现到底有没有Override某个特定的方法，但它不需要知道，需要程序员在类中显式指定

  1. .equals 和 == ：
    “==”是一个位比较，比较两者的每一个bit位， 对于java而言，string a =“Hello”, b=”Hello” , a==b是不成立的(a，b的对象地址不同)

.equals 是一个对象接口的方法（Object， java任意类的父类），在自定义类之中实现.equals之后，可以比较各种各样的对象
(可能需要用getClass()函数)
例如一个数组的equals可以这么定义：

@Override
public boolean equals(Object other){
    if(this.getClass() != other.getClass()){
      return false;  // 如果不同类则认为不能比较
    }
    if(this.size() != other.size()){ return false;}
    for(o : this){
        if(!other.contains(o)){
           return false;
        }
   }
   return true;
}

public Iterator<T> iterator();
Iterator<Integer> seer = ArrayObj.iterator();
while(seer.hasNext()){
   func(seer.next())  // for循环内的操作

也就是说，本质上迭代器需要满足的条件是：
有 .Iterator()方法.hasNext()方法和.next()方法，每迭代一次，迭代器走到下一个元素处，并返回前一个元素的值

如何实现可迭代自定义类呢？

  1. java的高阶函数
    在java之中高阶函数得间接做，因为java一切都是对象，但没有“函数”对象，所以我们需要把函数包装在对象之中，成为某个对象的方法，一般是用接口实现，例：
public interface TenX{
    public int funcX(int x);
}

public class IntFunction implements TenX{
    public int funcX(int x){return 10*x;}
}


public static int HOF_doTwice(IntFunction f, int x){
    return f.funcX(f.funcX(x));
}

在java之中，我们没有c++那样的运算符重载，但我们有.equals 和 .compareTo, 使用规范的泛型接口代替可能出现问题的运算符重载，这是更安全的做法

java提供的接口：
public interface Comparator {
public int compare(T o1, T o2); // return 0 if ==, >0 if > ,<0 if <
}

只需要

9. java的string
java在处理string时用了一种特殊的方式，没有“变量字符串”，只有常量字符串

例如：string a = “{” + “Ciallo!”+”}”， a会被构建三次，如果不断地使用+, 字符串就会不断的析构和构建，每次的构建不是在上一次的基础上在末尾加的，都是重新构建

这样对安全性有好处，但牺牲了一些效率，一种提升效率的方法是使用java内置的StringBuilder 类

StringBuilder SB = new StringBuilder("{");
SB.append("ciallo!");
SB.append("}");
System.out.println(SB.toString());
