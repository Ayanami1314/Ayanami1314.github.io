ts: 带静态类型的js

配置ts

tsc –init

tsconfig.json

重要配置

{

“compilerOptions”: {

“target”: “es2016”, // 指定编译器

“module”: “commonjs”,

“rootDir”: “./“,

……

“outDir”: “./“, // 编译完成的js文件位置

“removeComments”: true,

“noEmitOnError”: true,

}

tsc默认行为是编译root下面的所有.ts文件，如果上面改了tsconfig.json，编译rootDir

只需要tsc即可

debug ts

  1. 取消tsconfig.json 中的 sourceMap（源代码映射）的注释
  2. 配置debug里面launch.json，注意tsc: build如果开了中文包需要改成tsc: 构建(逆天)

ts的优点

还可以方便的和IntelliSense集成，对不同类型的对象提供不同的代码补全
