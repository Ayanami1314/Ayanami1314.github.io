调试理论

核心：程序是一个状态机，其状态是内存M，寄存器R的结合{M，R}，通过每一条指令转移状态

一个错误的程序Bug/fault，在经过一定的指令数之后，出现了理论状态与实际状态不符合的情况（Error）, 此时开始，可以被检测到，再经过一定的指令数（可能很长很长甚至没有）后，程序会崩溃形成一个直接可见的failure

程序状态：

​ |fault |Error |failure

right–>right–>…right–>error–>error–>error–>…

只有在right–>error这一步，才能被gdb调试出来，其他的right–>right, error–>error都是无法使用单步调试调出bug的

如何从failure定位fault?

printf: 抽取程序状态主要部分，可视化

二分printf, 寻找right–>error这一步，找到error处，进而回溯fault

更进一步？避免二分的麻烦

写好充足的assert等，让Error尽早变成failure

放到pa之中，即为Assert(), Log(), panic()函数的使用

IO和显卡

OS眼中的外设：三（或更多）个寄存器+一个程序

三个典型寄存器：外设状态state, 外设交换数据data，外设命令command

外设一般有自己的微型cpu，内存，芯片等，在OS眼中外设只是一些接口（软件上还附带了如何使用这些接口的驱动文件），OS只负责给外设发指令（指令集和发什么指令，什么时候发指令由外设的驱动提供），之后外设自己处理指令，并通过接口告诉OS

两种IO模式：memory-mapped IO & port-mapped IO (MMIO&PMIO)

mmap：在内存中保留一段地址空间用于存放外部设备

外设提供寄存器—>OS将其映射到某个内存地址—>之后如果其它程序要读写外设（当然它们只知道读写那个内存地址）—>OS转发给外设寄存器—>外设执行命令，OS使用polling或者interrupt的方式获取命令结果

OSTEP的伪代码（polling方式）

While (STATUS==BUSY)
    ; // wait until device is not busy 
Write data to DATA REG
Write command to COMMAND REG
    (device do something)
While (STATUS==BUSY)
    ; // wait until device is done

interrupt方式需要在中断表之中添加表项（现代的中断控制器集成在cpu之中，管理中断优先级和屏蔽等）

mmap好处是对硬件要求低，坏处提高了软件代码的编写难度

pmap: 在cpu上开出不同的IO端口

此时，IO的指令不再是和内存相同的write, read，而变成了特殊cpu指令；IO的存储也和内存分开，变成独立的空间

优点：省内存，缺点：耗cpu引脚

2023.11.6 正则表达式

正则：描述文本匹配的代码

最简单的正则：hi 匹配 hi

如果精确匹配hi，那么需要在前后加上\b，\b代表单词的开头或结尾, 是一个占位符，不代表空格或者换行（前后有一个空格能识别出这是一个单词即可）

\bhi\b.*\Ayanami\b 表示匹配{hi(若干个字符)Ayanami}

大概这样

\bhi\b.*\bayanami

hi,ddd ayanami

.表示匹配任意字符（除了换行），*表示前一个字符.可以重复任意次

\d ：0-9的一位数字

\d\d\d\d-\d\d\d\d\d\d\d\d 0579-12345678

重复简化语法

\d{4}-\d{8}

其他元字符

\s:匹配任意空白符

\w：数字、字母、下划线、（甚至可能支持汉字）

-D\w*=\w*\s

抓取make之中=的预定义宏-D

需要注意在这个时候，如果使用\s或者\b前后包裹起来，由于文本之中的-D可能是连着的，而两个连着的之间的空格只匹配一次，会漏匹配

抓-I

-I[a-zA-Z0-9\/]*

^匹配字符串开始

$匹配字符串结束

5-12位数字

^\d{5,12}$ (如果不用^$只能保证匹配的是数字，但不能保证数字连续)

对于\d{5,12}而言，使用这样的方法就只能保证字符串里包含5到12连续位数字，而不是整个字符串就是5到12位数字。

字符转义

元字符或者\， \\, \*,\.

重复

代码/语法 说明
* 重复零次或更多次
+ 重复一次或更多次
? 重复零次或一次
{n} 重复n次
{n,} 重复n次或更多次
{n,m} 重复n到m次

字符集

[aeiou]匹配a,e,i,o,u

[0-9a-zA-Z]

分支条件, 多种匹配取其一（有截断，先匹配左边）

|分割

\(\d{4}\)-\d{8}|\d{4}-\d{8}

反义：

  • \W：不是字母数字下划线

  • \S: 不是空白字符

  • \D: 不是数字

  • \B：不是单词的开始或结束

  • [^x]：不是x的任意字符

  • [^aeiou] ： 不是aeiou的任意字符

\S+ 不包含空白符的字符串

分组：多字符重复

多字符重复需要将子表达式套上小括号

(\d{1-3}.){3}\d{1-3} 简易ip地址匹配

但存在问题，ip之中三位是0-255

只能用很呆的方法

(2[0-4]\d|25[0-5]|[01]?\d?\d\.){3}(2[0-4]\d|25[0-5]|[01]?\d?\d)

后向引用：

子表达式会拥有组号，从左到右1，2，… 组号0是整体的表达式

后向引用用于重复某种分组匹配, 可以用\1指定第一组，\2指定第二组，etc

例如

\b(\w+)\b\s\1\b

用于匹配shit shit或者oh oh这样的连续重复单词，这里的\1是前面1组匹配的最终结果而不是字面表达式的复制

指定组名

(?”GroupName”\w+) 指定了这个组名为GroupName

引用时使用\k<GroupName>

零宽断言

断言：当断言为假的时候停止匹配

(?=exp) 断言：匹配的位置后面是exp

如：\b\w+(?=ing\b) 捕获singing, dancing等等，匹配结果为sing, danc

(?<=exp)断言：匹配位置的前面是exp

如：(?<=\bpre)\w+\b 捕获prereading 等等，匹配结果为reading

(?!exp)断言：匹配的位置后面不是exp

用于处理[^u]虽然达成了不是u的效果，但是要求必须存在一个字符的side effect

\b\w+\q(?!u)\w*\b 包含字母q，且后面不是u的单词

(?!<exp)断言：匹配的位置前面不是exp

\b(?!<_)usr\w*\b: 匹配usr而不匹配_user

(?<=<(\w+)>).*(?=</\1>): 匹配<html></html>格式

注释：（?#comment）

可以（?#cm1 #cm2 #cm3）

贪婪匹配和懒惰匹配：

通常情况：多是贪婪匹配，即匹配尽可能长的文本

a.*b 匹配aabab时，会匹配aabab全体

有时候我们需要匹配尽可能短的文本（懒惰匹配）

a.*?b 匹配aabab就会匹配aab和ab

懒惰限定符号：

  • *？
  • +？
  • ?? 重复0或1次，但尽量少重复
  • {n, m}?
  • {n,}?

平衡组和递归匹配：

抓取嵌套结构

(?’group’exp) : 把匹配的exp命名为group，并压入堆栈

(?’-group’exp): 匹配到exp时，堆栈上弹出group捕获内容，若栈空，则匹配失败

(?(group)yes|no):如果栈上有group,那么匹配yes，否则匹配no

(?!): 断言，由于没有后续的匹配部分?=exp，exp为空，必然失败

应用，匹配嵌套<div>

​ )+ # 1-n个左括号

​ [^<>]*

​ )+

​ (?(bracket)(?!)) # 如果bracket栈不空，即左右括号数量不匹配，则执行失败断言(?!)

压缩和软件设计哲学
2023/10/16

软件设计哲学：

让代码变得复杂的有两个主要因素：

Obscurity and dependency

为了避免Obscurity, 我们需要命名有意义的变量和抽象，比如用枚举代替奇怪的条件常数
为了避免dependency，我们需要避免重复，要做到“对一个bug，只修正一个方法”

管理复杂度：

  1. 编程时要更多地在“战略上”而不是“战术上”思考

  2. 我们无法一次处理完所有复杂度，所以我们需要隐藏复杂度

  3. build things deep: 最好的模块是使用起来简单易懂，但能够执行复杂操作的模块（不要让接口直接和底层绑定，尽可能少让一个模块的内部信息“泄露”到另一个模块）

  4. 警惕“时间分解”，事物发生的顺序并不是一个编程的好顺序，从整体上俯瞰是一个更好的做法

1. 拓扑排序

对于任意的节点i,j 如果存在从i到j的一条边，则j应该排序在i的后面
满足这样性质的排序叫拓扑排序

存在拓扑排序的条件：图是无环的

构建拓扑排序的方法：
对节点引入两个量，入度和出度，代表进入节点和从节点发出的边条数

首先，由于图是无环的，必然可以找到一个入度为0的点（“起点”）
抛去起点和与起点相连的边之后，子图必然有一个新“起点”

不断找这样的“起点”就得到了拓扑排序

算法是O（V+E）的

  1. 计算所有点的入度 O（V+E）
  2. 维护一个队列，包含所有入度为0的点，每次除去一个点和其邻边，并将相邻节点的入度减一（无向图）O（V+E）
